express-mysql-restfulAPI

Posted on 2018-12-27

【服务诞生原因】测试人员、开发人员常使用postman进行接口测试，但postman自身无法对mysql数据库进行操作，为了方便测试人员日常使用postman进行接口功能测试、接口自动回归测试，可以通过postman调用操作数据库清除测试数据。其次行业中借助postman调用数据库，使用比较多的是xmysql，但xmysql有个缺点，启用xmysql服务需要写死数据库账号密码，一旦接口地址外泄，对测试结果又一定影响。由此诞生express-mysql-restfulAPI，我为其命名为omysql

你可以从这里下载使用express-mysql-restfulAPI

Version 版本

V1.0.0

Run Environment 运行环境依赖

1.安装配置node环境

node 10及以上

参考

mac

2.所运用到的依赖包及版本

“body-parser”: “^1.18.3”

“express”: “^4.16.4”

“mysql”: “^2.16.0”

“multer”: “^1.4.1”

Deployment steps 部署步骤

  1. npm install //Installation node operation environment 安装node运行环境

  2. npm server.js //Start 启动，windows写个bat批处理开机启动，linux写个shell脚本开机启动

├── README.md // help

├── server.js // server

├── mysql_demo.js // mongdb

请求方法：POST

“host”:”127.0.0.1”,
“user”:”root”,
“password”:”12345678”,
“port”:”3306”,
“query”:”CREATE DATABASE Express_API_DB;”
}
